部落冲突复制阵型教程

发表于 2021-03-27 更新于 2025-04-24 分类于 其它

说明

使用阵型链接的方式有两种：

  1. 使用官方的网站，如果链接打不开就用下面这种方式
  2. 使用我开发的网站，复制阵型的原理和官方网站一样，不用担心会被封号

使用官方的网站

在小程序内复制到阵型的链接后，打开手机的浏览器，在浏览器中打开阵型的链接，然后点击中间的《打开部落冲突》按钮即可将阵型复制到游戏中，复制失败先看最下面的注意事项，如果不行请换一个浏览器试试

使用我开发的网站

无法直接打开阵型的链接，再使用本方法

网站的地址为：https://coc.zoutl.cn

点击上面网站的链接打开网址后，点击网站最上面的《教程》，然后根据提示来操作就行了，在教程页面输入链接复制阵型是不需要登录的

注意事项

一定要保证游戏在后台运行，也不要切后台时间太久，太久就会要重新加载游戏，大概率会复制不成功

国服阵型链接和国际服的阵型链接是不通用的，也就是说国服玩家不能使用国际服的阵型链接，反之亦然

如果发现阵型链接失效或者复制出来的阵型和图片不一样，请点击阵型图片下面的《纠错》按钮向我提交错误，如果我验证阵型链接有误，会返还你的复制次数

家乡可以复制和你等级差一级的阵型，例如：你现在是10本，你除了可以复制10本的阵型外，还可以复制9本和11本的阵型，但是夜世界和都城不适用于这个规则

如果你既玩国服又玩国际服，在手机浏览器复制阵型时，如果提示你选哪个应用打开，请不要勾选《不再提醒》或默认打开方式之类的东西，否则该浏览器就无法既复制国服阵型又复制国际服阵型了

解决：Tomcat运行项目时Quartz定时任务执行两次

发表于 2021-03-15 更新于 2022-04-26 分类于 运维

原因

在Tomcat的配置文件conf/server.xml中：

其中<Host>告诉Tomcat，在启动的时候加载webapps下的所有项目工程文件，<Context>又让Tomcat再加载了一遍（一般情况下配置<Context>，主要是由于想域名访问时将工程名去掉的原因配置），这种情况下会导致工程中的quartz定时被两次触发，执行两次

解决办法

将Tomcat的配置文件conf/server.xml修改为：

autoDeploy：是否允许自动部署，默认值是true，即表示 Tomcat 会自动检测appBase目录下面的文件变化从而自动应用到正在运行的 Web 应用程序
deployOnStartup=”false”，表示Tomcat服务器启动时, 不会自动发布appBase目录下所有的Web应用

会出现的问题

但是这样虽然解决了Quartz的执行两次的问题，但也存在另一个问题——无法解压war包。就是war包放到webapps目录下，不会自动解压生成项目文件夹，你所看到的项目文件夹是历史文件夹。所以，这是一个不断修改conf/server.xml的过程。当需要解压文件夹的时候，修改成true，解压完成后，改成false，记得重启Tomcat

PS.

MySQL数据库中text字段长度不够

发表于 2021-03-12 更新于 2022-04-26 分类于 数据库

问题描述

text类型是可变长度的字符串，最多65535个字符，有时候会超过其长度

解决办法

可以把字段类型改成MEDIUMTEXT（最多存放16777215个字符）或者LONGTEXT（最多存放4294967295个字符）
